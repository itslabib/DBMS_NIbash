\chapter{Introduction \& Overview}

\section{Introduction}
Nibash is a centralized, real-time web platform designed for the comprehensive management of residential buildings. The system bridges the gap between traditional manual property management and modern automated solutions. It provides distinct portals for building owners, residents, local service providers and administrators, offering role-based access control. 

\section{Video Demonstration}
\begin{tcolorbox}[colback=bglight, colframe=secondary, title=Project Demo]
The complete video demonstration for this project can be accessed at the following link:\\
\href{https://bit.ly/nibash-zero_nf}{\textbf{\textcolor{accent}{Video Link (https://bit.ly/nibash-zero\_nf)}}}
\end{tcolorbox}

\section{Motivation}
Managing residential buildings traditionally involves fragmented data management. This fragmentation leads to data inconsistency, communication gaps, and administrative delays. Moreover, paper-based visitor approvals are prone to human error and pose security risks. 

\vspace{0.5em}
The motivation behind Nibash is to consolidate these fragmented operations into a unified digital ecosystem. By leveraging AI-based CCTV surveillance, automated billing, biometric gate face scans, and comprehensive resident information tracking, the platform aims to significantly enhance both security and operational efficiency.

\subsection{Benchmark Analysis}
Compared to existing solutions, Nibash performs highly favorably in customizability, automation and real-time security.
\begin{itemize}[label=\textcolor{secondary}{\textbullet}]
    \item\textbf{CCTV Polling Latency:} Low-latency polling via 4 FPS JavaScript image fetches, providing near real-time feeds without heavy buffering.
    \item \textbf{Biometric Processing:} Utilizes lightweight OpenCV at the gate terminal for rapid resident face recognition, securely processing entries in under 2 seconds.
    \item \textbf{ALPR Processing Speed:} The Python-based EasyOCR server rapidly scans vehicle plates upon entry, instantly logging data and tracking available parking slots.
    \item \textbf{Automated Threat Detection:} Background CCTV processors instantly detect unknown faces and trigger real-time security alerts directly to the owner dashboard.
    \item \textbf{Financial Automation:} Seamless integration with DESCO APIs and the SSLCommerz payment gateway allows for instant utility calculations and real-time payment verifications.
\end{itemize}

\chapter{System Features \& Modules}

\section{Core Modules \& Portals}

We designed Nibash with a highly compartmentalized architecture consisting of distinct portals tailored to the specific needs of building owners, residents, and local service providers.

\subsection{Owner Portal}
The Owner Portal acts as the central command center for building administration.
\begin{itemize}[label=\textcolor{primary}{\textbullet}]
    \item \textbf{Interactive Dashboard:} A centralized overview displaying live building statistics, resident counts, quick-action widgets, and recent notifications..
    \item \textbf{Resident \& Apartment Management:} Provides a comprehensive interface to add, edit, view, and remove residents, as well as handle apartment assignments seamlessly.
    \item \textbf{CCTV Surveillance Dashboard:} A real-time camera wall integrating with IP cameras and mobile devices for live streaming, complete with a capture gallery and motion detection alerts.
    \item \textbf{User Entry Monitoring:} Allows owners to log manual entries, track all visitor activity, and monitor guest overstays effectively.
    \item \textbf{Community Hub \& Messaging:} A dual-layer communication system featuring building-scoped announcements (Community Hub) and a direct peer-to-peer messaging platform.
    \item \textbf{Automated Billing System:} End-to-end invoice generation tools with integrated SSLCommerz payment gateway and custom PDF templates for rent and utility collection.
\end{itemize}

\subsection{Resident Portal}
The Resident Portal focuses on convenience, transparency, and self-service.
\begin{itemize}[label=\textcolor{secondary}{\textbullet}]
    \item \textbf{Personal Dashboard:} An aggregated view of personal notices, recent activity, and quick access to essential services.
    \item \textbf{Biometric Guest Passes:} Allows residents to pre-register guests by providing their photo, enabling temporary facial recognition access at the security gate.
    \item \textbf{Billing \& Utilities Tracking:} Direct access to monthly invoices, payment history, DESCO electricity recharge records, and consumption graphs.
    \item \textbf{Smart Parking Management:} An integrated ALPR parking module allowing residents to view slot availability, register license plates (scanned via EasyOCR), and track booking statuses.
    \item \textbf{Platform Support Chatbot:} A helpful assistant integrated with the Gemini API to answer common questions and guide residents on using the portal.
    \item \textbf{Resident Profile \& Biometrics:} Enables residents to manage personal details, list emergency contacts, and enroll their own facial data for automated personal gate access.
\end{itemize}

\subsection{Security \& Administration}
This module handles building access and monitoring.
\begin{itemize}[label=\textcolor{accent}{\textbullet}]
    \item \textbf{Biometric Gate Access:} Uses a Python and OpenCV script to scan faces at the entry gate and log access records.
    \item \textbf{ALPR Parking System:} Uses EasyOCR to read vehicle license plates and authorize parking access.
    \item \textbf{CCTV Motion Processing:} A background script that checks camera feeds to detect movement or unknown faces and logs them in the system.
    \item \textbf{Emergency Console:} A control panel for administrators to view emergency logs or manually override gate controls.
    \item \textbf{Gate Kiosk:} A terminal placed at the entrance where unannounced guests can register and wait for resident approval.
\end{itemize}

\chapter{Database Architecture}

\section{Database Design Approach}
The Nibash database follows a relational design model utilizing MySQL. It is normalized up to the Third Normal Form (3NF) to reduce data redundancy and maintain data integrity.

\chapter{Implementation Details}

\section{Some Key SQL Queries}
The following SQL queries demonstrate the backend implementation of some core features.

\subsection{Fetching Owner Dashboard Statistics}
\begin{lstlisting}[caption=Dashboard Analytics Query]
SELECT 
    (SELECT COUNT(*) FROM users WHERE role = 'tenant') AS total_residents,
    (SELECT COUNT(*) FROM apartments WHERE status = 'occupied') AS occupied_apartments,
    (SELECT SUM(total_amount) FROM bills WHERE status = 'Paid' AND month = 'June') AS revenue,
    (SELECT COUNT(*) FROM guests WHERE entry_time >= CURDATE()) AS today_guests;
\end{lstlisting}

\subsection{Retrieving Service Provider Schedule}
\begin{lstlisting}[caption=Service Provider Schedule with Multi-table Joins]
SELECT 
    pb.booking_date, pb.time_slot, pb.status,
    up.full_name AS resident_name, up.phone AS resident_phone,
    b.building_name, a.apt_number
FROM provider_bookings pb
JOIN users u ON pb.resident_id = u.id
LEFT JOIN user_profiles up ON u.id = up.user_id
LEFT JOIN apartment_assignments aa ON u.id = aa.user_id AND aa.is_active = 1
LEFT JOIN apartments a ON aa.apt_id = a.id
LEFT JOIN buildings b ON a.building_id = b.id
WHERE pb.provider_id = ? AND pb.booking_date >= CURRENT_DATE
ORDER BY pb.booking_date ASC, pb.time_slot ASC;
\end{lstlisting}

\subsection{Retrieving Recent CCTV Captures}
\begin{lstlisting}[caption=CCTV Capture Retrieval]
SELECT c.id, c.image_path, c.detection_type, c.timestamp, d.camera_name 
FROM cctv_captures c 
JOIN cctv_devices d ON c.camera_id = d.id 
WHERE d.building_id = ? 
ORDER BY c.timestamp DESC LIMIT 12;
\end{lstlisting}

\newpage

\subsection{Fetching Monthly Bills with Payment Tracking}
\begin{lstlisting}[caption=Monthly Bill Breakdown with Payment Status]
SELECT 
    b.bill_number, b.month, b.year, b.issue_date, b.due_date,
    b.subtotal, b.tax, b.total_amount, b.paid_amount, b.status,
    p.transaction_id, p.payment_method, p.payment_date
FROM bills b
LEFT JOIN payments p ON b.id = p.bill_id
WHERE b.resident_id = ? AND b.apt_id = ?
ORDER BY b.year DESC, b.month DESC;
\end{lstlisting}

\subsection{Fetching Community Posts with Author Details}
\begin{lstlisting}[caption=Community Posts Retrieval with Comment Counts]
SELECT cp.id, cp.title, cp.content, cp.created_at,
       up.full_name AS author_name, a.apt_number, 
       cc.category_name,
       (SELECT COUNT(*) FROM community_comments WHERE post_id = cp.id) AS comment_count
FROM community_posts cp
JOIN users u ON cp.user_id = u.id
LEFT JOIN user_profiles up ON u.id = up.user_id
JOIN apartments a ON cp.apt_id = a.id
JOIN community_categories cc ON cp.category_id = cc.id
WHERE cp.status = 'published'
ORDER BY cp.is_pinned DESC, cp.created_at DESC;
\end{lstlisting}

\chapter{Conclusion \& Future Work}

\section{Limitations}
\begin{itemize}[label=\textcolor{accent}{\textbf{--}}]
    \item \textbf{Network Dependency:} The CCTV motion processor relies heavily on a stable local network. Network drops between IP cameras and the central server can cause temporary stream disconnections.
    \item \textbf{Hardware Constraints:} Biometric face recognition processing time increases linearly with the number of enrolled faces unless indexed effectively, requiring adequate CPU/GPU power.
\end{itemize}

\section{Future Work}
\begin{itemize}[label=\textcolor{primary}{\textbullet}]
    \item \textbf{Mobile Application Integration:} Developing native Android and iOS applications to complement the web platform, specifically for pushing instant push notifications.
    \item \textbf{Advanced AI Models:} Upgrading the motion detection processor from standard OpenCV background subtraction to deep learning models (like YOLO) for precise human classification.
    \item \textbf{IoT Smart Home Integration:} Expanding the platform to control smart locks directly from the resident dashboard.
\end{itemize}

\section{Conclusion}
The Nibash Smart Building Management Platform successfully demonstrates the integration of traditional property management workflows with modern, AI-driven security and payment technologies. By centralizing operations into a cohesive web application, Nibash provides an intuitive, real-time, and secure environment for both property owners and residents.
